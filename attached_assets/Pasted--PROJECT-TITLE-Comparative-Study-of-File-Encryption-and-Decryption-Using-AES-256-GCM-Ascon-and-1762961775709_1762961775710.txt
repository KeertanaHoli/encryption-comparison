📦 PROJECT TITLE:
Comparative Study of File Encryption and Decryption Using AES-256-GCM, Ascon, and ChaCha20-Poly1305

🎯 PURPOSE:
Build a Python Flask web app that encrypts and decrypts FILES (not text) using AES-256-GCM, Ascon-128a, and ChaCha20-Poly1305. Each operation is recorded in a local blockchain ledger. The app auto-generates a benchmark graph that updates dynamically after each operation — no separate file uploads required.

⚙️ TECH STACK:
Python 3.11+, Flask, pycryptodome, ascon, matplotlib, pandas, SQLite or JSON, HTML+Bootstrap (Replit Free plan friendly).

🧰 FEATURES:
1️⃣ ENCRYPTION & DECRYPTION
- Upload any file → choose algorithm → enter passphrase.
- Derive 256-bit key using PBKDF2 (SHA-256 + random salt).
- Encrypt file → embed salt, nonce, tag, and algorithm in header.
- Download encrypted file; decrypt by re-uploading same file + passphrase.
- Automatically log encryption/decryption time, file size, and algorithm to blockchain.

2️⃣ PERFORMANCE BENCHMARK
- Automatically generates bar chart comparing AES, Ascon, and ChaCha20 based on encryption/decryption time and file size.
- Chart built from blockchain data (no extra uploads).
- Matplotlib base64 chart updates after each operation.
- “Clear Chart” button resets visualization.

3️⃣ BLOCKCHAIN LEDGER
- Append-only ledger in ledger.db or ledger.json.
- Each block stores: index, timestamp, prev_hash, tx_hash, algorithm, file_name, file_hash, nonce_b64, tag_b64, salt_b64, file_size_bytes, enc_time_ms, dec_time_ms.
- `/verify` endpoint validates blockchain integrity.
- Interactive HTML table view.

📊 METRICS:
| Algorithm | Enc Time (ms) | Dec Time (ms) | File Size (KB) |

📁 STRUCTURE:
app.py → Flask routes  
crypto_utils.py → encryption/decryption logic  
blockchain.py → blockchain handling  
benchmark.py → chart generation  
templates/ → index.html, benchmark.html, ledger.html  
storage/encrypted/, storage/decrypted/  
ledger.db or ledger.json  
requirements.txt, README.md

📦 REQUIRED PACKAGES:
flask, pycryptodome, ascon, matplotlib, pandas

⚡ FUNCTIONAL NOTES:
- Works directly on uploaded files, no text input.
- Benchmark uses timing and file size from blockchain logs.
- Passphrase only used for PBKDF2 key derivation.
- Metadata stored inside encrypted file header.
- Blockchain stores only metadata, not keys or passphrases.
- “Clear Chart” resets chart but keeps ledger.
- Fully compatible with Replit Free plan limits.

🧠 HINTS:
AES: AES.new(key, AES.MODE_GCM, nonce)
Ascon: ascon.encrypt(key, nonce, data, ad) / ascon.decrypt(...)
ChaCha20: ChaCha20_Poly1305.new(key, nonce)
PBKDF2: PBKDF2(passphrase, salt, dkLen=32, count=200000)
Use time.perf_counter() for timing.
File header: algorithm_id | salt | nonce | tag | ciphertext
Flask routes: / (main), /benchmark, /ledger, /verify

⚠️ SECURITY NOTE:
Educational demo only — passphrases are never stored.
